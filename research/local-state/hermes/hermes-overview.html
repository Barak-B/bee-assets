1מה Hermes עושה — בשפה פשוטה

Hermes הוא סוכן AI חכם שיושב על המחשב שלך ועושה שלושה דברים מרכזיים:

אמקבל הודעות

דרך WhatsApp (פעיל) · מהטרמינל ישירות · מ-cron jobs מתוזמנים. בעתיד גם Email, Telegram, Discord.

בחושב ופועל

שולח את ההודעה למודל AI (DeepSeek V4 Pro ברירת מחדל), ה-AI מחליט מה לעשות, משתמש בכלים (טרמינל, browser, חיפוש קוד).

גמחזיר תשובה

מבצע את המשימה (כותב קובץ, מריץ סקריפט, מחפש מידע) ושולח לך חזרה את התוצאה ב-WhatsApp.

דוגמה מעשית: אתה שולח ל-WhatsApp "בדוק כמה אתרים סולאריים פעילים היום". Hermes מקבל את ההודעה, ניגש למסד הנתונים של BEE Operations, מריץ שאילתא, מסכם — ומחזיר תוך 5 שניות "486 אתרים פעילים, 99% מנוטרים".

3תזרים — הודעה אחת מקצה לקצה

מה קורה ברקע כשאתה שולח הודעת WhatsApp ל-Hermes:

אתה שולח הודעה ב-WhatsApp

הודעה מגיעה ל-WhatsApp Cloud → ל-bridge.js שמאזין על port 3000.

זמן: מיידי

סינון allowlist

הברידג׳ בודק אם המספר ברשימה המורשית (`WHATSAPP_ALLOWED_USERS`). אם לא — נדחה. אם כן — ממשיך.

קובץ: allowlist.js

Python Gateway קורא את ההודעה

הגייטוויי שואל את הברידג׳ "יש הודעות חדשות?" (long-poll על /messages) ומקבל אותה.

קובץ: gateway/platforms/whatsapp.py

בונה context

שולף מ-state.db את ההיסטוריה של השיחה הזו, טוען את ה-skills הרלוונטיים, מצמיד את הזהות שלך.

קובץ: agent/prompt_builder.py

שולח ל-LLM (DeepSeek V4 Pro)

שולח את כל הקונטקסט ל-API של DeepSeek. ה-AI חוזר עם תשובה או "תקרא לי לכלי X" (tool call).

קובץ: agent/transports/openai.py

לולאת כלים (tool loop)

אם ה-AI ביקש כלי — Hermes מריץ אותו (טרמינל, browser, read_file), מחזיר את התוצאה ל-AI, וחוזרים עד שה-AI מסיים. מקסימום 40 סבבים.

קובץ: run_agent.py

שומר את הסשן

כל ההודעות, ה-tool calls, התוצאות — נשמרים ב-state.db לשיחה הבאה.

DB: state.db (SQLite)

שולח תשובה חזרה

הגייטוויי שולח POST /send לברידג׳ עם התשובה. הברידג׳ מעביר ל-WhatsApp. אתה רואה את ההודעה בטלפון.

זמן כולל: 3-15 שניות

5יעדים — כמה רחוק להשלמה מלאה

8 יעדים אסטרטגיים שיהפכו את Hermes ל-Worker Bee מלא של B.E.E:

1. יציבות תמידית (Always-on)

50%

log rotation עובד ✓ · NSSM הותקן ✓ · אבל service טרם רשום ולא מופעל. דורש admin להפעלת auto-restart.

חסר: רישום שירותמאמץ: 5 דקות (admin)

2. ערוצי תקשורת מרובים

25%

WhatsApp פעיל ✓ · Terminal פעיל ✓ · אבל Email · Telegram · Discord · Slack — כולם לא מוגדרים. כל אחד = פשוט dot env + setup wizard.

חסר: 4 ערוציםמאמץ: 5-15 דקות לכל אחד

3. זיכרון מתמשך (Persistent Memory)

60%

state.db שומר היסטוריית סשנים ✓ · אבל memory plugin (vector store, knowledge graph) עדיין לא פעיל. Hermes לא זוכר עובדות בין סשנים שונים, רק בתוך אותה שיחה.

חסר: vector memoryמאמץ: 2-4 שעות

4. יעילות טוקנים

60%

תוקנו: max_turns 90→40, vision aux. נשאר: CLI sessions = 59% מהטוקנים מ-2.5% מהsessions (heavy!). חקירת deepseek-chat anomaly (169M tokens unexplained).

חסר: CLI patterns + anomalyמאמץ: 1-2 שעות

5. Auto-supervision (השגחה אוטומטית)

40%

log rotation מתוזמן ✓ · אבל אין metrics endpoint, אין alerts אם משהו נופל, אין health-ping cron. אם הברידג׳ ייפול ב-2 בלילה — אף אחד לא ידע עד הבוקר.

חסר: metrics + alerts + pingמאמץ: 3-4 שעות

6. סוכנים מתמחים (Level 2)

15%

יש 2 skills ייחודיים ל-B.E.E (barak-business-integrations, barak-activity-reporting). חסרים: hermes-solar, hermes-clients, hermes-finance, hermes-field. כל אחד = סוכן ייעודי עם credentials/scope משלו.

חסר: 4-5 specialized agentsמאמץ: חצי יום עבודה

7. Orchestrator (Level 3)

0%

בעתיד: bee-hive יקבל כל פנייה, יחליט איזה specialized agent מתאים, ינתב, יסכם תוצאות. אחת הדלת הקדמית, מאחור צוות סוכנים. עדיין לא קיים.

תלוי ב: יעד 6מאמץ: יום עבודה

8. צוות אוטומטי (Level 4)

0%

היעד הסופי: cron + webhooks → orchestrator → task-bus (inbox → working → outbox → done) → צוות. אתה לא בלולאה כל היום, רק review בסוף יום. עדיין שלב הרבה רחוק.

תלוי ב: יעדים 6+7מאמץ: יומיים
השלמה כוללת: 42% — חצי הדרך. הנתיב ברור: גלי עבודה (Waves) שכבר תוכננו, חלק כבר בוצעו.
