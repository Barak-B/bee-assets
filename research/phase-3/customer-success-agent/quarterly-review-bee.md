# quarterly-review-bee

## What it does

Auto-generates a presentation-ready QBR for an enterprise customer, synthesizing
their portfolio's financial + production + service metrics into a Hebrew narrative.

Anthropic's original is SaaS-shaped (deal pipeline, MRR). BEE adaptation focuses
on **solar portfolio performance**: production vs forecast, savings delivered,
service responsiveness, expansion opportunities.

## Parameters

```
--customer <id>        # e.g., rafael_solar
--quarter <YYYY-QN>    # default: previous quarter
--save-to <drive|local|both>
```

## 6-step workflow (Anthropic pattern, BEE content)

### Step 1 — Portfolio production performance

```
- For all customer's sites: total kWh produced this quarter
- vs engineering-agent forecast (performance_forecast)
- Performance ratio (actual / expected)
- Best + worst performing sites
- Savings delivered: kWh × tariff = ₪ saved for customer
- Compare QoQ + YoY
```

### Step 2 — Service health

```
- Alerts this quarter (count, severity breakdown)
- SLA performance: % met by tier
- Avg response + resolution time
- Open issues at quarter end
- Jobs completed at their sites
```

### Step 3 — Financial (with customer)

```
- Invoices issued + paid (from Invoice Maven)
- Payment timeliness
- Any maintenance contract revenue
- Account value YoY
```

### Step 4 — Opportunities

```
- Underperforming sites → repair/optimization upsell
- Sites near capacity → expansion potential
- Battery retrofit candidates (sites without storage)
- Maintenance contract renewal/upsell
```

### Step 5 — Hebrew narrative (500-800 words)

Sections:
- כותרת + תקופה
- ביצועי ייצור (production trajectory)
- חיסכון שנצבר (savings delivered)
- בריאות שירות (service health + SLA)
- הזדמנויות לרבעון הבא (opportunities)
- סיכונים (risks: underperformance, warranty expiries)
- קריאה לפעולה (next-quarter call to action)

Generated by Claude Sonnet (quality tier — customer-facing).

### Step 6 — Export

```
- Hebrew RTL PDF: qbr-<customer>-<YYYY-QN>.pdf
- Charts: production vs forecast, savings cumulative, SLA performance
- ASCII tables fallback if chart lib unavailable
- Upload to customer portal (Q73) + Drive
- Notify Barak: draft ready for review before sending
```

## MCP dependencies

- `bee-mcp`: getCustomer, listSites(customer_id), getSiteProduction, listAlerts
- `invoice-maven-mcp`: invoices + payments for the customer this quarter
- `engineering-agent`: performance_forecast for expected production
- `customer-pulse-bee`: health score + trend for the narrative
- `anthropic-docx-skill`: Hebrew PDF

## Approval gate

QBR is customer-facing → **Barak reviews + approves before it's sent or
portal-published**. Agent generates draft, notifies Barak, waits.

## Differences from Anthropic original

| Anthropic | BEE |
|---|---|
| QuickBooks P&L | Invoice Maven + BEE app per-customer financials |
| HubSpot deal metrics | Solar portfolio production metrics |
| MRR/churn focus | kWh delivered + savings + SLA focus |
| English | Hebrew RTL |
| Generic SMB | Solar enterprise portfolio |
| 20% revenue concentration flag | Site-level performance concentration |

## ROI

Eliminates ~6h manual QBR prep per customer per quarter. 5 tier_1 × 4 = 20
QBRs/year → ~100-120h saved. Plus: regular QBRs are a known retention driver
for enterprise accounts (the "QBR is where renewal is decided" — Anthropic
comparison §2).
